The AES (Rijndael)* Block Cipher   (*AES and Rijndael Can be use interchangeable)

            The AES (Rijndael)  block cipher is designed to use simple whole-byte operations. AES algorithm can support length of  key and block sizes of 128, 192, or 256 bits. Choosing length of key and block size would impact to the number of the round.

            AES (Rijndael) consist of several round transformation , transformation can be divided into three main round : initial round , nine round transformation (rounds) and final round that performed at the end of process(encryption /decryption).

            For decryption core that will be designed, initial round composed of three transformations :

Ø       Add_Round_Key Transformation

Ø       Invers Byte_Sub Transformation

Ø       Invers Shift_Row Transformation

And for nine round transformation composed of four transformation such as:

Ø       Add_Round_Key Transformation

Ø       Invers Mix_Column Transformation

Ø       Invers Byte_Sub Transformation

Ø       INvers Shift-Row Transformation

For the final round or at the end of decryption process just held Add_Round_Key transformation

Description of each transformation.

  • Byte substitution. Each 8-bit byte in the state is reversibly mapped into another byte (S-box).  For inverse Byte substitution each 8-bit byte in the state mapped into another byte (inverse S-box).
  • Shift-row. Row 0 is unmoved, Row 1, Row 2 and Row 3 are rotated left 1, 2 and 3 bytes respectively. For inverse Shift-Row Row 0 is unmoved . row1, row 2, and row 3 are rotated left 3,2,1 byte respectively   
  • Mix-column. Each column is regarded as a degree-3 polynomial over GF(2**8), and multiplied by the polynomial p(x) ='03'x**3 + '01'x**2 + '01'x + '02', modulo (x**4 + 1). For inverse mix-column is reverse of mix-column transformation and multiplied by polynomial that have relationship p(x)  d(x) = ‘01’

d(x) = '0b'x**3 + '0d'x**2 + '09'x + '0e'

  • Sub-key addition. The sub-key for the current round is XORed into the state.

Decryption process in AES algorithm ,each sub-key that used in add_Round_Key transformation was generated in reverse order to the encryption process. Because each sub-key composed of four Array W, the role to generated each byte in W  describe below:

For  (i +1)/4   0  so  W[i] = W[i-4] xor W[i-3]

For   (i+1)/4 = 0  so:

W[i]= W[i-4] xor ByteSub(S1 (W[i-6] xor W[i-7]))  xor (rcon[9-round]&’000000’)
